# Тестовое задание - системный аналитик

Репозиторий содержит выполненное тестовое задание на позицию системного аналитика.

## Состав задания

1. Анализ требований к функционалу корзины интернет-магазина.
2. Проектирование REST API для экрана выбора магазинов-партнёров.
3. Верхнеуровневая архитектура отправки PUSH-уведомлений в мобильное приложение.

## Краткое описание решения

### Задание 1

В первой части проведён анализ фрагмента ТЗ по функционалу корзины.  
Выделены логические противоречия, неоднозначности, дублирующие требования и неполные места.  
Также предложена исправленная версия требований и список уточняющих вопросов к PM / бизнес-заказчику.

### Задание 2

Во второй части описан REST API для получения списка магазинов-партнёров в мобильном приложении.  
Приведён пример `GET`-запроса, пример успешного JSON-ответа, описание полей ответа и возможные ошибки.

### Задание 3

В третьей части предложена верхнеуровневая архитектура отправки PUSH-уведомлений.  
Решение построено вокруг отдельного `Notification Service`, который принимает заявки от бизнес-сервисов, получает device token пользователя, выбирает шаблон уведомления, отправляет PUSH через FCM/APNs и сохраняет результат отправки.

## Примечания

Диаграмма в третьем задании выполнена в формате PlantUML.  
Если GitHub отображает `.puml` файл как текст, его можно открыть в любом PlantUML-редакторе для просмотра схемы.
